Git Bash Here找不到应用程序

在使用Git时突然发现git命令不能用了QAQ，关了连Git Bash Here都打不开，估计是因为把Git的存储位置换了个地方的原因叭，解决方案如下：

1、win+R打开命令行窗口，输入”regedit”打开注册表

2、根据路径查找
“HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Directory\background\shell\git_shell\command” 和 “HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Directory\shell\git_shell\command”，将其数值数据里面的路径修改为现在的“git bash.exe”的路径

3、同第二步查找路径，将其git_shell改为git_gui，找到这两个路径，并将里面的路径修改为现在的“git gui.exe”的路径

4.关闭注册表，接着在文件夹右键选择Git Bash Here，发现终于可以打开了

git commit后，怎么撤回commit？

输入
git reset --soft HEAD^
撤销成功！不删除工作空间改动代码，仅撤回commit操作，不撤销git add .

git reset --soft HEAD~2
撤回两次commit

git reset --mixed HEAD^ git reset HEAD^
不删除工作空间改动代码，撤销commit，并且撤销git add . 操作

git reset --hard HEAD^
删除工作空间改动代码，撤销commit，撤销git add . （完成此操作后，则恢复到上次commit状态）

git commit --amend
若commit注释写错，此命令则可以改动注释，此时会进入默认vim编辑器，修改注释完毕后保存即可。

如何将Git库推送到远端仓库？

Git 库推送到远端仓库一般有两种情况：

  1. 本地新建的仓库；

  2. clone 的别人的项目到本地。

第一种（就本人个人）：
在经过。。。。一系列操作后，没有push到远程仓库，发现git出问题了（第一种 救命），弄好之后，又又又进行
git add .
git commit -m "first commit"
git remote add origin https://github.com/xxx.git
git push -u origin master
一系列操作才成功
注：这里关联远程仓库HTTPS链接很有可能不能用啊（本人就是，试了n次），改用SSH成功

第二种：
假设在远端建立的新仓库： ‘https://github.com/xxx.git'
1、如果你是 clone 别人的仓库到本地，那么 git 肯定会存在别人关联的仓库，要先把这个关联仓库删除才行，执行命令：

git remote remove origin —— 删除当前 Git 库关联的远程仓库地址

2、把你自己的远程库和本地库进行关联，执行命令：

git remote add origin https://github.com/xxx.git —— 关联自己的远程仓库地址

3、关联好后，把本地所有内容推送到远程库上：

git push -u origin master —— 推送内容到远程仓库

注意：当我们第一次推送 master 分支时，加上了 -u 参数， Git 不但会把本地的 master 分支内容推送到远程新的 master 分支，还会把本地的 master 分支和远程的 master 分支关联起来。
